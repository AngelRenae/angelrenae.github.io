Semantic Versioning

  1. Home
  2. Tools of the Trade
  3. Semantic Versioning

Keeping track of the production product is just as important as keeping track of your code progress using GitHub. This can be achieved using semantic versioning, a way of keeping track of what version the software is in. It is formatted as MAJOR.MINOR.PATCH, where it starts at 1.0.0 and each value is incremented depending on the change. This is all based on backwards compatibility. MAJOR is incremented if the new version is not backwards compatible, MINOR is incremented if there is substantial change but itâ€™s still backwards compatible, and PATCH is incremented if there are just bug changes.
